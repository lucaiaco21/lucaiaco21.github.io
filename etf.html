The Problem

Why ETFs are hard to predict.

ETF prices are driven by a combination of macro trends, sector momentum, and the idiosyncratic behavior of individual funds. Standard statistical models fail to capture non-linear dependencies across time. The goal here was to build a sequence model that could learn temporal patterns in price movements — across multiple ETFs simultaneously — using both numerical features and categorical embeddings.

The target variable is the next-day log return (Close_log_diff), which stabilizes variance across ETFs with very different price scales and makes the model inherently aware of relative movement rather than absolute levels.

Exploratory Data Analysis

Understanding the data first.

Before modeling, I profiled each ETF individually — checking price distributions, return volatility, stationarity via ADF tests, and autocorrelation structure via ACF/PACF plots. The dataset covers multiple ETF symbols across different sectors and asset categories, creating a naturally imbalanced panel.

Key EDA findings: price series were non-stationary (as expected), log-differencing achieved stationarity for all symbols, and seasonality decomposition revealed weekly periodicity in volume but not in returns.

Data Quality

Handling nulls & outliers.

Missing values appeared primarily at the start of each ETF's history — where certain features hadn't yet accumulated enough data (e.g. rolling averages). These were forward-filled within each symbol group, then any remaining leading nulls were dropped. Cross-symbol contamination was avoided by always grouping before imputing.

Outliers in log returns were handled via IQR-based clipping per symbol — extreme values caused by earnings events or macro shocks were capped rather than removed, preserving the sequence structure needed for LSTM training.

Python Outlier Clipping per Symbol 
# IQR clipping per symbol to preserve sequence continuity
def clip_outliers(g, col, factor=3.0):
    q1, q3 = g[col].quantile([0.25, 0.75])
    iqr = q3 - q1
    g[col] = g[col].clip(q1 - factor*iqr, q3 + factor*iqr)
    return g

df = df.groupby("symbol", group_keys=False).apply(
    clip_outliers, col="Close_log_diff"
)
Feature Engineering

Building the right inputs.

  • 01 Log Returns & Normalization Raw closing prices were transformed to log differences (log(Close_t / Close_t-1)) to produce a stationary, scale-invariant target. All sequential features were normalized using MinMaxScaler fitted on the training set only — preventing data leakage into validation and test.
  • 02 Rolling Technical Indicators A set of rolling window features was engineered per symbol: moving averages (5d, 10d, 20d), rolling volatility, volume momentum, and relative strength proxies. These provide the model with local trend context within each sequence.
  • 03 Growth Since Creation A symbol-level growth feature was constructed by mapping each ETF to its reference price at inception. The ratio Close / reference_price − 1 encodes long-run momentum that short rolling windows cannot capture.
  • 04 Categorical Embeddings (Symbol & Category) Each ETF symbol and asset category was label-encoded and passed as a separate integer input to a learned Embedding layer inside the model. This allows the network to implicitly learn latent representations of each fund's behavior — separate from the time-series signal.
Python Sequence Construction 
def create_sequences(df, window=30):
    X_seq, X_symbol, X_cat, y = [], [], [], []

    for sym, g in df.groupby("symbol"):
        g = g.reset_index(drop=True)
        for i in range(window, len(g)):
            seq    = g.loc[i-window:i-1, feature_cols].values
            target = g.loc[i, "Close_log_diff"]
            X_seq.append(seq)
            X_symbol.append(g.loc[i, "symbol_id"])
            X_cat.append(g.loc[i, "category_id"])
            y.append(target)

    return (
        np.array(X_seq,    dtype=np.float32),
        np.array(X_symbol, dtype=np.int32).reshape(-1, 1),
        np.array(X_cat,    dtype=np.int32).reshape(-1, 1),
        np.array(y,        dtype=np.float32).reshape(-1, 1)
    )
Model Architecture

How the LSTM is built.

The model takes three inputs: a 30-step sequence of numerical features, a symbol ID, and a category ID. The sequence goes through two stacked LSTM layers. The symbol and category IDs are passed through Embedding layers, flattened, and concatenated with the LSTM output. A Dense layer with L2 regularization processes the merged representation before the final scalar prediction.

Hyperparameters — LSTM units, embedding dimensions, dropout rate, L2 strength, and learning rate — were tuned via Keras Tuner's RandomSearch across 20 trials, monitoring validation loss with early stopping at patience 8.
